## 1. 项目概述

### 1.1 背景
XX市安全生产责任保险信息云平台（安责险平台）是面向安责险全业务链条的数字化管理系统，覆盖投保、保全、理赔、预防服务、政府监管、财务结算等核心业务。3.0版本在2.x版本基础上，重构前端架构，引入AI能力，统一多端（PC管理端 + 小程序端）交互体验。

### 1.2 目标
- **可用性**：支持5类角色端口，满足政府监管、保险公司、服务机构、运营平台、企业用户的差异化需求
- **AI能力**：OCR保单识别、AI理赔识别、AI行业画像、AI报表生成、AI智能客服
- **数据闭环**：从投保 → 保单管理 → 预防服务 → 理赔 → 结算 全流程数字化
- **监管合规**：支持数据上报、数据质控、数据直报，满足上级平台对接需求

### 1.3 架构形式
- **PC端**：单文件多视图SPA（`index.html`），内联CSS/JS，无外部CDN依赖，Ant Design蓝白配色系统
- **小程序端**：独立`xcx.html`原型，后续微信小程序开发
- **后端**：RESTful API（本文档不含后端接口规范，需另行出接口文档）

## 3. 用户角色与端口

系统共5个端口（Portal），登录后弹出端口选择器，按账号权限展示可选端口。

| 端口 | data-portal | 角色描述 | 特有功能 |
|------|-------------|----------|---------|
| 运营管理员端 | `operation` | 平台运营方，拥有全部权限 | 企业匹配、AI报表中心、业务员业绩统计、系统运维、结算规则配置 |
| 政府端 | `government` | 政府安监部门，只读+审批 | 任务检查（发起方）、数据上报 |
| 保险业务员端 | `insurance` | 保险公司业务员 | 批单管理、OCR识别、费用结算 |
| 服务专家端 | `service` | 安全服务机构专家 | 集中服务列表、企业自主排查（协助） |
| 企业用户端 | `enterprise` | 投保企业用户 | 企业自主排查（发起）、微信绑定 |

### 3.1 端口权限切换规则
- 同一账号可绑定多个端口，登录后弹选端口
- 切换端口：侧边栏底部"切换端口"按钮 → 重新弹出端口选择弹窗（不退出登录）
- 当前端口标识显示在侧边栏顶部Logo旁（小字badge）

## 4. 全局交互规范

### 4.1 SPA路由机制
- 所有视图为 `.page` div，默认 `display:none`
- 激活视图追加 `.active`（`display:block`）
- 菜单点击 → `showPage(pageId)` 函数切换视图
- URL hash 可选同步（`#module-submodule`）

### 4.2 侧边栏交互
- **展开宽度**：240px，**收起宽度**：64px（仅显示icon）
- 顶部搜索框：输入关键词实时过滤菜单项，匹配高亮 `.highlight`，不匹配追加 `.hidden-by-search`
- 二级菜单：点击父菜单展开/收起子菜单（Accordion式，同一时间只展开一个父菜单）
- 菜单角标：红色圆形badge显示待处理数量
- 底部常驻：切换主题 / 切换端口 / 退出登录

### 4.3 顶部Header
- 高度：56px，背景：`rgba(10,22,40,0.95)`，底部边框：`rgba(0,212,255,0.2)`
- 左侧：面包屑导航（`/`分隔，当前页加粗白色）
- 右侧：通知bell（红色badge数字）+ 用户头像 + 用户名 + 下拉箭头
- 下拉菜单：个人设置 / 切换端口 / 退出登录

### 4.4 通知系统
- 铃铛图标点击展开通知面板（Dropdown）
- 通知类型：系统通知 / 业务提醒 / 待办事项
- 已读/未读状态，支持"全部已读"批量操作
- 未读总数显示在badge

### 4.5 弹窗规范
- **Modal弹窗**：宽度480px~800px不等，居中显示，含mask蒙层
- **Drawer抽屉**：从右侧滑入，宽度800px~1000px，不遮挡侧边栏
- 关闭方式：右上角×按钮 / 点击mask / ESC键（抽屉不响应mask点击关闭）
- 表单弹窗底部：取消按钮（灰色）+ 确认按钮（蓝色主色）
- 危险操作弹窗底部：取消按钮 + 确认删除按钮（红色 `#ff4d4f`）

### 4.6 表格通用规范
- 表头背景：`#fafafa`，字体颜色：`#666666`
- 奇偶行不区分颜色（统一白色）
- Hover行：`#f5f5f5`
- 选中行：`#e6f7ff`（蓝色浅底）
- 固定列：checkbox列固定左侧（z-index:5），操作列固定右侧（左侧有box-shadow）
- 超长文字：`overflow:hidden; text-overflow:ellipsis; white-space:nowrap`，hover展开tooltip
- 数字列右对齐（`.col-number`）
- 分页：左侧显示"共X条"，右侧分页按钮，支持切换每页条数（10/20/50/100）

### 4.7 状态标签（Tag）规范
| 样式类 | 颜色 | 适用状态 |
|--------|------|---------|
| `tag-green` | 绿色 `#52c41a` | 有效、在保、已完成、正常 |
| `tag-red` | 红色 `#ff4d4f` | 无效、脱保、已拒绝、异常 |
| `tag-yellow` | 橙色 `#faad14` | 待处理、预警、临期 |
| `tag-gray` | 灰色 `#666` | 草稿、已撤回、停用 |
| `tag-blue` | 蓝色 `#1890ff` | 进行中、审核中、待确认 |

### 4.8 按钮规范
| 按钮类型 | 样式 | 适用场景 |
|---------|------|---------|
| 主要按钮 `.btn-primary` | 蓝色实心 | 新增、确认、提交 |
| 默认按钮 `.btn-default` | 白色边框 | 查询、重置、导出 |
| 危险按钮 `.btn-danger` | 红色实心/边框 | 删除、驳回 |
| 文字按钮 `.btn-link` | 无边框蓝色文字 | 表格操作列内查看/编辑 |
| 图标按钮 `.btn-icon` | 无边框icon | 收起/展开/刷新 |

## 5. 通用组件规范

### 5.1 筛选栏组件
- 位于页面顶部，白色卡片内
- 布局：行内flex，每项标签+控件，右侧"查询"主按钮+"重置"默认按钮
- 控件类型：文本输入框 / 下拉选择框 / 日期范围选择器 / 级联选择器
- 查询触发：点击查询按钮 或 按Enter键
- 重置：清空所有筛选条件，重新加载全量数据

### 5.2 统计卡片组件（`.pc-stat-card`）
- 宽度：1/4等分（grid 4列）
- 高度：约100px
- 内容：顶部小标签（13px）+ 大数字（28px bold）+ 底部趋势说明（12px）
- 颜色：蓝色/绿色/橙色/紫色渐变背景，白色文字

### 5.3 数据表格+分页组件
- 表格需支持：列显示/隐藏切换（右上角icon）、列宽拖拽（可选）
- 分页默认每页20条
- 批量操作：勾选后顶部出现批量操作浮层（含已选X条文字 + 操作按钮）

### 5.4 上传组件
- 支持拖拽上传 + 点击上传
- 格式限制显示在上传区域下方（红色小字）
- 上传后显示文件名+大小+删除按钮
- 上传进度条（蓝色）

### 5.5 Rich文本/预览组件
- 用于服务报告、排查记录等内容展示
- 支持图片预览放大

## 8. 数据字典

### 8.1 枚举值定义

**保单状态**：
- `PENDING`：待起保
- `ACTIVE`：在保
- `EXPIRING`：临期（30天内到期）
- `EXPIRED`：已脱保
- `VOID`：已失效
- `CANCELLED`：已退保

**服务状态**：
- `NOT_STARTED`：未开始
- `IN_PROGRESS`：进行中
- `REVIEWING`：待审核
- `COMPLETED`：已完成
- `REJECTED`：已驳回

**理赔状态**：
- `REPORTED`：已报案
- `ASSESSING`：定损中
- `PROCESSING`：理赔中
- `SETTLED`：已结案
- `WITHDRAWN`：已撤案

**AI监测结果**：
- `NORMAL`：正常
- `WARNING`：预警
- `ABNORMAL`：异常

**任务状态**：
- `PENDING_ASSIGN`：待分配
- `IN_PROGRESS`：执行中
- `PENDING_REVIEW`：待审核
- `COMPLETED`：已完成
- `OVERDUE`：已逾期
- `REJECTED`：已驳回

**签章状态**：
- `PENDING`：待审核
- `APPROVED`：已批准
- `REJECTED`：已拒绝
- `WITHDRAWN`：已撤回

**账单状态**：
- `DRAFT`：草稿
- `PENDING_CONFIRM`：待确认
- `CONFIRMED`：已确认
- `DISPUTED`：争议中
- `SETTLED`：已结算

**数据上报校验状态**：
- `PENDING`：待上报
- `SUCCESS`：上传成功
- `FAILED`：上传失败
- `PARTIAL`：部分成功

**隐患等级**：
- `MAJOR`：重大（红色标注）
- `GENERAL`：一般（橙色标注）
- `MINOR`：轻微（黄色标注）

**日志级别**：
- `INFO`：信息（蓝色）
- `WARN`：警告（橙色）
- `ERROR`：错误（红色）

### 8.2 状态Tag颜色映射
所有模块状态颜色遵循统一规范：
- 绿色（green tag）→ 成功、正常、完成类状态
- 蓝色（blue tag）→ 进行中、待处理类状态
- 橙色（yellow tag）→ 预警、待审核类状态
- 红色（red tag）→ 失败、异常、紧急类状态
- 灰色（gray tag）→ 草稿、停用、撤回类状态
